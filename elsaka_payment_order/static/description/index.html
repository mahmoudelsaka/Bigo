Elsaka Payment / Receiving Order Features



1- Making work-flow for payment or receiving money to manage treasury by creating a receipt order or a payment order

2- A work-flow is applied on order to be approved form operation manager and financial manager

3-After receipt is approved, it will be added automatic to selected cash register according to payment or receiving according to its type also currency

4- It should be used for miscellaneous or customer / supplier payment

5- Printing Receipt Order, Payment Order and Receipt Voucher , Payment Voucher



Receipt Order / Payment Order



Partner (Optional): May be Customer or Supplier or Employee, or empty in case of miscellaneous amount

Type: Receipt, in case of order to receive money, Payment in case of order to pay cash

Amount: Should be amount number

Memo: Text to mapped to Cash register line

Due date: Data to mapped to cash register line and effect journal entries

Currency: It will let order to mapped to open cash register with same currency

After Order is approved,



It will be mapped as positive amount in case of receipt and as negative amount in case of payment



From that point, Cashier can reconcile it with invoice or expenses account.



Also System will generate journal accordingly

User can print payment or receipt orders

User Can print Receipt Voucher and payment Voucher after transaction is done
